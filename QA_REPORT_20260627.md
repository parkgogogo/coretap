## 详细问题

### BUG-001：OCR 在主屏可见文字上失败

严重程度：P1

影响：

- `assert text` 不可靠。
- `wait text` 不可靠。
- `tap text` 无法完成“精确文字点击”主流程。
- 这会阻塞不走 VLM 的快速文字定位路径。

复现命令：

```bash
coretap --format json --backend device --device 00008101-000875523E30001E assert text --text "搜索" --timeout-ms 3000
coretap --format json --backend device --device 00008101-000875523E30001E assert text --text "Watch" --timeout-ms 3000
coretap --format json --backend device --device 00008101-000875523E30001E tap text "搜索" --dry-run
coretap --format json --backend device --device 00008101-000875523E30001E wait text --text "搜索" --timeout-ms 3000
```

实际结果：

- `assert text "搜索"` 返回 `TEXT_ASSERTION_FAILED`。
- `assert text "Watch"` 返回 `TEXT_ASSERTION_FAILED`。
- `tap text "搜索"` 返回 `TEXT_TARGET_NOT_FOUND`。
- `wait text "搜索"` 返回 `TEXT_ASSERTION_FAILED`。

证据：

```text
artifacts/coretap/run_20260627T094545Z_fa89ef1b/assert.png
artifacts/coretap/run_20260627T094545Z_fa89ef1b/assert-001.tsv
artifacts/coretap/run_20260627T094633Z_7ce4dd98/assert.png
artifacts/coretap/run_20260627T094655Z_07c5c113
artifacts/coretap/run_20260627T094718Z_47a967e8
artifacts/coretap/run_20260627T094740Z_9b3f8ed6
```

观察：

- 截图里“搜索”和 “Watch” 都可见。
- Tesseract 输出 token 质量很差，没有识别出目标文本。
- 这不是语言包缺失问题，`ocr check` 已确认 `chi_sim+eng` 可用。

### BUG-002：`type` 返回成功但真实输入不精确

严重程度：P1

影响：

- `type` 作为 UI 自动化输入文本的主能力目前不可靠。
- 对测试脚本来说，命令返回成功但实际输入错误，风险高于直接失败。

复现命令：

```bash
coretap --format json --backend device --device 00008101-000875523E30001E tap point --space normalized --x 0.5 --y 0.835
coretap --format json --backend device --device 00008101-000875523E30001E type "hello@example.com"
coretap --format json --backend device --device 00008101-000875523E30001E screenshot --label qa-type-after
```

预期结果：

```text
hello@example.com
```

实际结果：

```text
hello2恶心啊没票了。 co m
```

证据：

```text
artifacts/coretap/run_20260627T094842Z_07ffa1af/qa-type-after.png
```

观察：

- JSON 返回 `ok: true`。
- 返回结果里 `typedCharacters: 17`。
- 实际输入受当前中文输入法/键盘状态影响，字符被输入法改写。

### BUG-003：`screenshot --out` 触发 userspace tunnel 单例冲突

严重程度：P1/P2

影响：

- 指定输出路径截图不可用。
- 失败后 daemon 仍运行，但 CoreDevice sessions 被清空，容易影响后续用例稳定性。

复现命令：

```bash
coretap --format json --backend device --device 00008101-000875523E30001E screenshot --out artifacts/coretap/qa-out.png
```

实际结果：

```text
COREDEVICE_SCREENSHOT_FAILED
Persistent CoreDevice screenshot failed: a userspace tunnel is already active in this process (PyTCP's stack is a process-global singleton; only one userspace tunnel per process is supported)
```

证据：

```text
artifacts/coretap/qa-out.png 未生成
```

观察：

- 普通 `screenshot --label` 可以成功。
- `--full-size` 可以成功。
- `--max-long-side 1024` 可以成功。
- 目前只在 `--out` 用例上复现。

### BUG-004：`replay` 未复用 1368 压缩输入，耗时偏高

严重程度：P2

影响：

- `replay` 可用但性能偏差明显。
- replay 和 locate/tap target 的输入图处理路径不一致，后续排查模型稳定性时可能产生误导。

复现命令：

```bash
coretap --format json replay artifacts/coretap/run_20260627T095048Z_52ed1fbe
```

实际结果：

- replay 成功。
- recorded/replayed 坐标接近，漂移约 `x: 3.375px`、`y: -2.436px`。
- replay 耗时约 `12.6s`。
- 返回结果里没有 `modelInput` 字段。

对照命令：

```bash
coretap --format json model run --image artifacts/coretap/run_20260627T095048Z_52ed1fbe/source.model-input.png --target "the Health app icon"
```

对照结果：

- `model run` 使用 1368 压缩输入，耗时约 `3.9s`。

证据：

```text
artifacts/coretap/run_20260627T095048Z_52ed1fbe
```

### BUG-005：normalized 输入时 `screenshotPx` 字段容易误解

严重程度：P3

影响：

- JSON 返回可读性差。
- 调试坐标时容易把 normalized 值误认为真实截图像素坐标。

复现命令：

```bash
coretap --format json --backend device --device 00008101-000875523E30001E tap point --space normalized --x 0.5 --y 0.5 --dry-run
coretap --format json --backend device --device 00008101-000875523E30001E drag --from 0.5,0.75 --to 0.5,0.25 --dry-run
```

实际结果：

```json
"screenshotPx": {"x": 0.5, "y": 0.5}
```

观察：

- 在没有 frame/width/height 的情况下，当前返回值不是真实像素。
- 更清晰的行为可能是省略 `screenshotPx`、返回 `null`，或明确标记 frame 未知。

### BUG-006：`model install --dry-run` 参数语义不安全

严重程度：P2

影响：

- CLI 暴露了 `--dry-run`，但 install 路径没有使用它。
- 当前测试环境里模型已安装，所以没有副作用；如果模型未安装，可能会发生用户以为不会发生的安装动作。

复现命令：

```bash
coretap --daemon off --format json model install --dry-run
```

实际观察：

- 返回成功。
- 代码路径里 `command_model` 对 `install` 调用的是 `model_install(args.profile, force=args.force)`，没有传递或处理 `args.dry_run`。

相关位置：

```text
coretap/cli.py
```

## 后续建议

建议优先级：

1. 先修 `type` 输入准确性，因为命令返回成功但真实输入错误，自动化风险最高。
2. 再修 OCR 可靠性，否则 `tap text` / `assert text` / `wait text` 这一整条快速文字路径不可用。
3. 修 `screenshot --out` 的 userspace tunnel 冲突，避免指定输出路径导致 daemon session 被清空。
4. 统一 `replay` 和 VLM 主流程的图片压缩路径。
5. 调整 normalized 坐标返回字段，减少调试误解。
6. 修正 `model install --dry-run` 的参数语义。
